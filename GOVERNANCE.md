# Governance

This project uses a lightweight benevolent-dictator model.

## Roles

- **Maintainer**: Dax Davis. Final decision maker on scope, design, and releases.
- **Contributors**: anyone who opens an issue or a pull request.

## How decisions are made

Most decisions happen in issues and pull requests in the open. The maintainer
reviews and merges contributions, sets the roadmap, and cuts releases. When
there is disagreement, the maintainer makes the call and explains the reasoning.

The guiding principle is scope discipline: this plugin scans Apify-reachable
job boards and feeds curated roles into the career-ops pipeline. Changes that
keep it simple, sandboxed, and human-in-the-loop are the easy yes.

## Changing this model

If the project grows enough to need more structure, this document will be
updated in a pull request before any change takes effect.
